# Алгоритм бота v2 (без LLM)

Новая версия бота с **минимальным использованием AI**: только распознавание голоса (Whisper). Всё остальное — детерминированные правила.

---

## Меню

- **Старт** (`/start`) — приветствие и короткая подсказка.
- **Добавить задачу** (`/add`) — переводит в режим «следующее сообщение = текст задачи».
- **Список** (общий список задач) — аналог `/tasks`: все активные задачи.

---

## Добавление одной задачи

Задачу можно добавить **двумя способами**.

### 1. Через команду «Добавить задачу»

1. Пользователь нажимает **«Добавить задачу»** (или пишет `/add`).
2. Бот отвечает: «Напиши или надиктуй задачу — следующее сообщение я сохраню как задачу».
3. Для пользователя выставляется **состояние «ожидаю задачу»** (в памяти: `user_id → awaiting_task`).
4. **Следующее** сообщение пользователя (текст или голос) считается **полным текстом задачи**:
   - если текст — берём его целиком;
   - если голос — сначала распознаём через **Whisper** (без LLM), затем полученный текст = текст задачи.
5. Из текста задачи **алгоритм** (без AI) извлекает:
   - дату: «сегодня», «завтра», «послезавтра», «в пятницу», «через N дней», дата вида ДД.ММ;
   - время: «в 10 утра», «в 14:00», «в 10:30» и т.п.
6. Если дата не указана — **автоназначение**: сегодня или завтра в зависимости от загрузки дня (лимит из настроек).
7. Задача сохраняется в БД с категорией «Другое», приоритет по умолчанию. Отправляется подтверждение (текст, срок, приоритет).
8. Состояние «ожидаю задачу» снимается.

### 2. Через фразу «Добавь / Создай / Запиши …»

1. Пользователь пишет или говорит одну фразу, например:
   - «Добавь купить молоко завтра»
   - «Создай задачу записаться к врачу в пятницу»
   - «Запиши позвонить маме»
2. Бот проверяет начало сообщения на **маркеры**: «добавь», «создай», «запиши», «добавить», «создать», «поставь задачу», «запланируй» и т.п.
3. Если маркер есть — **всё после маркера** считается текстом задачи (без вызова LLM).
4. Дальше — как в п.1: парсинг даты/времени алгоритмом, автоназначение при отсутствии даты, сохранение, подтверждение.
5. Состояние «ожидаю задачу» здесь не используется.

---

## Голосовые сообщения

- Голос обрабатывается **только через Whisper** (речь → текст).
- LLM для понимания или классификации **не вызывается**.
- После получения текста из Whisper он обрабатывается по тем же правилам, что и обычное текстовое сообщение:
  - если пользователь в состоянии «ожидаю задачу» — весь распознанный текст = задача;
  - иначе если текст начинается с маркера «добавь»/«создай»/«запиши» — убираем маркер, остаток = задача;
  - иначе весь текст считаем одной задачей (как будто после «добавить задачу»).

---

## Текстовые сообщения (не команды)

- Если пользователь в состоянии **«ожидаю задачу»** → всё сообщение = текст задачи, парсим дату/время, сохраняем, снимаем состояние.
- Иначе если сообщение **начинается с маркера** («добавь», «создай», «запиши» …) → убираем маркер, остаток = задача, парсим и сохраняем.
- Иначе → ответ: «Чтобы добавить задачу: нажми «Добавить задачу» и отправь текст или напиши «Добавь [задача]».» (без вызова AI).

---

## Что не использует LLM

- Классификация намерения (задача / не задача) — по состоянию и по маркерам.
- Извлечение текста задачи — обрезка префикса по маркерам или «следующее сообщение».
- Парсинг даты и времени — регулярные выражения и ключевые слова (как в текущем боте).
- Автоназначение даты — подсчёт задач по дням и лимит из настроек.
- Сохранение в БД, формирование списка задач, подтверждение — обычный код.

---

## Что использует внешний API (не LLM)

- **Whisper** (Groq/OpenAI и т.п.) — только преобразование голоса в текст. Дальше работает тот же алгоритм, что и для введённого текста.

---

## Итог

- **Один источник нестабильности убран**: нет зависимости от ответов LLM и лимитов на запросы к нему.
- Добавление одной задачи и общий список работают по чётким правилам; голос — только как способ ввода текста через Whisper.
