# Рутины: продуктовое проектирование

Документ в роли продукта: пользовательские истории, бэклог, триггеры распознавания и способ реализации.

---

## 1. Что такое рутины

**Рутины** — задачи, которые нужно выполнять на регулярной основе. Они не имеют одной конкретной даты: у них задаётся **регулярность** (ежедневно, каждый понедельник, раз в неделю и т.д.). Бот должен в нужный момент напоминать о них, включая в список задач на сегодня и в общие списки.

---

## 2. Пользовательские истории

**US-RT1. Записывать повторяющиеся дела**  
Я как пользователь хочу записывать рутины с заданной периодичностью, чтобы не держать их в голове и чтобы бот напоминал о них в нужный день.

**US-RT2. Рутины в плане на сегодня**  
Я как пользователь хочу видеть рутины, выпадающие на сегодня, в плане на сегодня («Что на сегодня?»), чтобы понимать полную картину дня вместе с разовыми задачами.

**US-RT3. Рутины в общем списке задач**  
Я как пользователь хочу видеть рутины в общем списке задач (например, в разделе «Рутины» или с пометкой регулярности), чтобы помнить, что они у меня есть, и при необходимости редактировать или отключать.

**US-RT4. Отдельный раздел «Рутины»**  
Я как пользователь хочу иметь отдельный пункт/команду «Рутины», где перечислены все мои рутины с указанием регулярности (каждый понедельник, раз в неделю, ежедневно), а не даты, чтобы управлять ими отдельно от разовых дел.

**US-RT5. Распознавание по фразам**  
Я как пользователь хочу создавать рутину естественной фразой («Записать рутину: поливать цветы каждый четверг», «Ежедневная: зарядка», «Еженедельная уборка по воскресеньям») или по команде, чтобы не искать отдельную форму — бот сам поймёт, что это рутина.

**US-RT6. Рутины в отчётах за неделю**  
Я как пользователь хочу видеть выполнение рутин в отчёте «Сделано за неделю» (с пометкой, что это рутина, и с регулярностью), чтобы оценивать, насколько стабильно я соблюдаю привычки.

**US-RT7. Раз в неделю без дня — случайный день**  
Я как пользователь хочу, чтобы если у рутины не задан конкретный день недели, но задана периодичность «раз в неделю» (или «еженедельно», «каждую неделю»), бот автоматически и рандомно назначал один день недели для показа рутины, чтобы рутина стабильно появлялась в плане один раз в неделю, а не каждый день.

**US-RT8. Несколько раз в неделю без дней**  
Я как пользователь хочу писать фразы вроде «массаж два раза в неделю» или «3 раза в неделю», чтобы бот сам выбрал дни недели с примерно равными промежутками и по возможности не перегружал дни, где у меня уже много других рутин.

## 3. Слова-триггеры: когда задача считается рутиной

Задача считается рутиной, если в тексте пользователя (или в явной команде) есть маркеры повторяемости. Ниже — список триггеров для распознавания и синонимы.

### 3.1 Команды / намерения

| Триггер | Пример |
|--------|--------|
| записать рутину | «Записать рутину: полив цветов» |
| добавить рутину | «Добавить рутину — зарядка» |
| рутина | «Рутина: проветрить спальню» |
| регулярн | регулярная задача, регулярно мыть полы |

### 3.2 Ежедневность

| Триггер | Пример |
|--------|--------|
| ежедневн | ежедневная зарядка, ежедневно поливать |
| каждый день | каждый день зарядка |
| каждый день | каждый день пить витамины |
| раз в день | раз в день капли |
| по утрам / по вечерам | по утрам зарядка (можно трактовать как ежедневно + time_of_day) |

### 3.3 Еженедельность (общая)

| Триггер | Пример |
|--------|--------|
| еженедел | еженедельная уборка |
| раз в неделю | раз в неделю мыть полы |
| каждую неделю | каждую неделю стирка |
| раз в неделю | уборка раз в неделю |

### 3.4 Конкретный день недели

| Триггер | Пример |
|--------|--------|
| каждый понедельник | каждый понедельник планерка |
| по понедельникам | по понедельникам стирка |
| каждую среду | каждую среду йога |
| по средам | по средам бассейн |
| каждый четверг | каждый четверг полив цветов |
| по четвергам | по четвергам уборка |
| каждый пятницу / каждую пятницу | каждую пятницу закупки |
| по пятницам | по пятницам массаж |
| каждую субботу | каждую субботу уборка |
| по субботам | по субботам рынок |
| каждое воскресенье | каждое воскресенье план на неделю |
| по воскресеньям | по воскресеньям отдых |
| в понедельник (в контексте повторения) | уборка в понедельник и четверг |
| пн, вт, ср, чт, пт, сб, вс | полив цветов чт, ср (короткая форма) |

### 3.5 Реже (опционально на будущее)

| Триггер | Пример |
|--------|--------|
| раз в месяц | раз в месяц оплата |
| каждые две недели | каждые две недели стрижка |

Рекомендация: на первом этапе реализовать **ежедневно** и **по дням недели** (пн–вс). «Раз в неделю» без указания дня можно трактовать как «ежедневно» или как рутину с одним выбранным днём по умолчанию (например, воскресенье).

## 4. Формат хранения регулярности

В БД уже есть поля `is_routine` (bool) и `repeat_day` (text).

- **repeat_day** хранит человекочитаемую регулярность, например:
  - `ежедневно` — показывать каждый день;
  - `пн` — только понедельник;
  - `вт,чт` или `вт, чт` — вторник и четверг;
  - `пн,ср,пт` — три дня в неделю.

Нормализация при сохранении: приводить к коротким кодам (`пн`, `вт`, …) для единообразия и маппинга на weekday (0–6). В интерфейсе отображать: «Каждый понедельник», «Каждый вторник и четверг», «Ежедневно», «Раз в неделю» (если решим хранить один день для «раз в неделю»).

## 6. Связь с текущим кодом

- **db.py**: `add_task(..., is_routine, repeat_day)`, `get_today_tasks` (уже фильтрует рутины по `_routine_matches_today`), `_ROUTINE_DAY_MAP` (пн–вс, ежедневно). При необходимости расширить маппинг синонимов дней («понедельник» → `пн`).
- **bot.py / bot_v2.py**: при добавлении задачи вызывать определение «это рутина?» и парсинг регулярности; передавать `is_routine` и `repeat_day` в `db.add_task`.
- **Список рутин**: отдельный хендлер (команда /routines или фраза «рутины», «мои рутины») — выборка `status='active' AND is_routine=TRUE`, форматирование с колонкой «Регулярность» вместо даты.
- **План на сегодня**: уже использует `get_today_tasks` — рутины, подходящие на сегодня, там есть; остаётся единая нумерация и визуальное выделение (🔁).
- **Отчёты**: в `_format_done_report_today` и `_format_done_report_week` для задач с `is_routine=True` (или по флагу в выполненной задаче) добавлять 🔁 и при желании подпись регулярности.

## 7. Алгоритм определения рутины (псевдокод)

```
def is_routine_and_repeat(task_text: str) -> tuple[bool, str | None]:
    text = normalize(task_text)
    # 1) Явные команды
    if "записать рутину" in text or "добавить рутину" in text or " рутина " in text:
        repeat = parse_repeat_from_text(text)  # ежедневно | пн | вт,чт | ...
        return True, repeat
    # 2) Ежедневность
    if any(kw in text for kw in ["ежедневн", "каждый день", "раз в день", "по утрам", "по вечерам"]):
        return True, "ежедневно"
    # 3) Еженедельно (общее)
    if any(kw in text for kw in ["еженедел", "раз в неделю", "каждую неделю"]):
        # Опционально: парсить день или по умолчанию "вс"
        return True, parse_weekday_or_default(text, default="вс")
    # 4) Конкретный день
    day = parse_weekday_from_text(text)  # "каждый понедельник" -> "пн", "по средам" -> "ср"
    if day:
        return True, day
    # 5) Несколько дней: "пн и чт", "вт, чт"
    days = parse_multiple_weekdays(text)
    if days:
        return True, ",".join(days)
    return False, None
```

Функция `parse_weekday_from_text` должна учитывать: понедельник/пн, вторник/вт, среда/ср, четверг/чт, пятница/пт, суббота/сб, воскресенье/вс, а также «каждый понедельник», «по понедельникам» и т.д.
