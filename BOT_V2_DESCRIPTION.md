# Описание бота версии 2

Документ описывает, что умеет бот v2, как он работает и какие тесты к нему написаны.

---

## Что умеет бот

Бот v2 — **облегчённый помощник по задачам**: только добавление одной задачи и просмотр списка. Без LLM: распознавание намерений и дат делается по правилам и состоянию; для голоса используется только Whisper (речь → текст).

### Меню

| Команда | Описание |
|--------|----------|
| **Старт** (`/start`) | Приветствие и подсказка по командам. |
| **Добавить задачу** (`/add`) | Режим «следующее сообщение = задача». Или фраза «Добавь [задача]», «Создай [задача]», «Запиши [задача]». |
| **Список задач** (`/tasks`) | Все активные задачи с *нумерацией*, группировкой по дате, человекочитаемыми датой/временем. Перед показом просроченные задачи переносятся на сегодня. |
| **План на сегодня** (`/today`) | Задачи на сегодня (с нумерацией). |
| **Отметить выполнение** (`/done`) | Подсказка: выполнить по номеру («Отметь 3») или по названию («Выполни купить молоко»). |
| **Сделано сегодня** (`/done_today`) | Отчёт: выполненные задачи за сегодня. |
| **Сделано за неделю** (`/done_week`) | Отчёт: выполненные задачи за последние 7 дней. |

### Добавление задачи

- **Одна задача за раз** — либо следующим сообщением после `/add`, либо одной фразой вида «Добавь …» / «Создай …» / «Запиши …».
- Из текста задачи **автоматически** извлекаются:
  - **Дата**: «сегодня», «завтра», «послезавтра», «в пятницу», «через 3 дня», дата вида 15.04 или 10/05.
  - **Время**: «в 10 утра», «в 14:00», «в 7 вечера» и т.п.
- Если дата не указана — бот сам назначает день (сегодня или завтра) по загрузке и настройке лимита задач на день.
- После сохранения бот присылает подтверждение: текст задачи, категория (пока «Другое»), срок, приоритет (по умолчанию 5/10).

### Голосовые сообщения

- Голос **сначала распознаётся через Whisper** (речь → текст).
- Дальше тот же алгоритм, что и для текста: если было нажато «Добавить задачу» — весь распознанный текст считается задачей; если фраза начинается с «добавь»/«создай»/«запиши» — остаток после маркера; иначе весь текст сохраняется как одна задача.
- LLM для понимания или классификации **не используется**.

### Перенос просроченных задач

- При открытии списка задач или плана на сегодня бот переносит активные задачи с датой выполнения в прошлом на *сегодня* (однократно за запрос).

### Выполнение задач

- **По номеру:** «Отметь 3», «Выполни 1» — номер из текущего списка задач (нумерация 1, 2, 3…).
- **По названию:** «Выполни купить молоко», «Отметь задачу полить цветы» — поиск по подстроке. Если найдено несколько задач, бот показывает их с номерами и просит уточнить (написать номер).
- Синонимы: отметь, отметить, выполни, выполнить, сделай, сделать, заверши, готово, сделано (в т.ч. голосом).

### Текст задачи в верхнем регистре

- При добавлении задачи её название сохраняется и отображается в **верхнем регистре**.

### Текстовые и голосовые команды (синонимы)

- **Список задач:** «покажи список задач», «все задачи», «мои задачи», «план», «задачи» и т.п.
- **План на сегодня:** «план на сегодня», «задачи на сегодня», «что на сегодня».
- **Добавить задачу:** «добавить задачу», «новая задача» — включает режим «следующее сообщение = задача».
- Выполнение по фразе — см. выше (отметь/выполни).

### Что бот не делает в v2

- Нет рутин, истории, настроек, удаления/редактирования по фразе.
- Нет разбиения одного сообщения на несколько задач и нет определения категории/приоритета через AI.

## Как работает бот (логика)

### Запуск

- По умолчанию в `bot_replit.py` запускается **v2** (`bot_v2`). Чтобы запустить старую версию (v1 с LLM), задаётся переменная окружения **`BOT_VERSION=1`**.
- Используются те же ключи, что и у старого бота: `TELEGRAM_BOT_TOKEN`, `DATABASE_URL`, для голоса — ключ к Whisper (например `GROQ_API_KEY`).

### Состояние «ожидаю задачу»

- После команды `/add` для пользователя (по `telegram user id`) в памяти выставляется флаг: **следующее** входящее сообщение (текст или голос) трактуется как **полный текст одной задачи**.
- После обработки этого сообщения флаг сбрасывается.

### Обработка текстового сообщения

1. Если сообщение — команда (`/start`, `/add`, `/tasks`) → выполняется команда.
2. Если пользователь в состоянии «ожидаю задачу» → весь текст = задача → парсинг даты/времени → автоназначение даты при необходимости → сохранение в БД → подтверждение → сброс состояния.
3. Иначе если текст **начинается с маркера** («добавь», «создай», «запиши», «добавить», «создать», «поставь задачу», «запланируй», «закажи», «нужно», «надо») → отрезается маркер (и при необходимости «новую задачу») → остаток = текст задачи → парсинг → сохранение → подтверждение.
4. Иначе бот отвечает подсказкой: нажать «Добавить задачу» и отправить текст или написать «Добавь [задача]».

### Обработка голосового сообщения

1. Файл голоса отправляется в **Whisper** → получаем текст.
2. Дальше тот же сценарий, что для текста: состояние «ожидаю задачу» → весь текст задача; иначе маркер в начале → остаток задача; иначе весь текст = одна задача. Парсинг даты/времени и сохранение общие с текстом.

### Парсинг (модуль `task_parsing.py`)

- **Дата**: ключевые слова «сегодня», «завтра», «послезавтра», «через N дней», названия дней недели («пятницу», «среду» и т.д.), явная дата `ДД.ММ` или `ДД/ММ`. Результат — `YYYY-MM-DD` или `None`.
- **Время**: «в 10:30», «в 10 утра», «в 7 вечера» и т.п. Результат — `HH:MM` или `None`.
- **Текст задачи после маркера**: отрезается один из префиксов и при необходимости «новую задачу», возвращается остаток (или весь исходный текст, если маркера нет).
- **Проверка маркера**: функция `starts_with_add_marker(text)` — да/нет.

Автоназначение даты делается в `bot_v2` по БД: лимит задач на день из настроек пользователя, выбор ближайшего дня с числом задач ниже лимита.

## Тесты

К боту v2 написаны **юнит-тесты на логику парсинга** — без Telegram и без БД. Проверяется только модуль **`task_parsing`**, который используется в `bot_v2`.

### Файл тестов

- **`tests/test_task_parsing.py`** — тесты для функций:
  - `parse_due_date(text, today=None)`
  - `parse_due_time(text)`
  - `extract_task_text(user_text)`
  - `starts_with_add_marker(text)`

Для дат в тестах задаётся фиксированный день (понедельник 2 марта 2026), чтобы результаты не зависели от текущей даты.

### Что покрыто тестами

| Группа | Что проверяется |
|--------|-----------------|
| **Парсинг даты** | «сегодня», «завтра», «послезавтра»; «через N дней»; день недели; явная дата; отсутствие даты → `None`. |
| **Парсинг времени** | «в 10:30», «к 12 завтра», «в 10 утра» и т.д.; отсутствие времени → `None`. |
| **Извлечение текста задачи** | После «Добавь», «Создай», «Запиши» — остаток; обрезка «новую задачу». |
| **Маркер добавления** | «Добавь …», «создай …», «нужно …» → `True`; иначе → `False`. |
| **Маркер выполнения** | «Отметь 3», «выполни купить молоко», «сделано» → `True`. |
| **Извлечение цели выполнения** | «отметь 3» → (3, ""); «выполни купить молоко» → (None, "купить молоко"). |
| **Форматирование списка** | Нумерация, группировка по дате, человекочитаемые дата/время. |
| **План на сегодня, отчёт** | _format_today_list, _format_done_report. |

### Запуск тестов

Из корня проекта:

```bash
python -m pytest tests/ -v
```

Краткий прогон:

```bash
python -m pytest tests/ -q
```

Остановка на первой ошибке:

```bash
python -m pytest tests/ -x
```

Перед доработкой фич тесты рекомендуется прогонять, чтобы не сломать текущее поведение парсинга.
