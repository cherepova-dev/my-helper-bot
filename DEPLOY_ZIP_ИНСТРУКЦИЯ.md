# Деплой через Zip (резервный способ)

> **Основной способ деплоя — Git push на JustRunMy.App, см. `JUSTRUNMYAPP_GIT_DEPLOY.md`.**
> Сборка Docker-образа в обоих случаях идёт **в облаке JustRunMy**, локальный Docker не нужен.
>
> Этот документ — на случай, когда `git push justrunmyapp main:deploy` стабильно отвечает
> `No matching nodes (preffer node not found)` и в течение нескольких часов в **Deployment**
> не появляется новый Docker Image. Часто задача проходит сама позже — сначала просто подожди и
> повтори пуш, и только потом используй Zip.

## Шаг 1: Собрать архив

В папке проекта выполни:

```powershell
.\make_deploy_zip.ps1
```

Появится файл **`bot-deploy.zip`** с актуальным кодом (включая установку меню команд при старте).

## Шаг 2: Загрузить в JustRunMy.App

1. Зайди в [JustRunMy.App](https://justrunmy.app) → своё приложение **interesting_carver**.
2. Открой раздел **Deployment** (или тот, где раньше загружала Zip).
3. Выбери **Upload** / **Загрузить Zip** и укажи файл **`bot-deploy.zip`**.
4. Дождись сборки и перезапуска приложения.

## Шаг 3: Проверить

В **Diagnostics** после перезапуска должны появиться:

- `DEPLOY_STAMP: 20260228-menu-v3`
- `MENU: команды установлены при старте (7 пунктов)` или сообщение об ошибке
- `Бот запущен (MVP). [меню при старте]`

В Telegram открой чат с ботом → нажми по полю ввода → открой список команд (кнопка «/» или меню слева). Должны быть команды: Начать, Все задачи, План на сегодня и т.д.

---

Переменные окружения (TELEGRAM_BOT_TOKEN, GROQ_API_KEY, DATABASE_URL и т.д.) задаются в настройках приложения на JustRunMy.App и при смене деплоя на Zip **не сбрасываются** — проверь, что они по-прежнему указаны.
