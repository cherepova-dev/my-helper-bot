# Бэклог продукта: отчёты и категории

Документ в роли продуктового менеджера и архитектора. Содержит пользовательские истории и техническое проектирование.

---

## 1. Отчёты («Сделано сегодня» и «Сделано за неделю»)

### 1.1 Пользовательские истории (формат: я как пользователь хочу…)

**US-R1. Быстрая обратная связь за день**  
Я как пользователь хочу в конце дня быстро увидеть, что я успел сделать сегодня, чтобы чувствовать удовлетворение от результата и не терять ощущение прогресса.

**US-R2. Отчёт за сегодня в удобном виде**  
Я как пользователь хочу, чтобы отчёт «Сделано сегодня» показывал задачи с датой и временем выполнения, чтобы вспомнить контекст (что сделал утром, а что вечером).

**US-R3. Группировка по категориям**  
Я как пользователь хочу видеть выполненные задачи, сгруппированные по категориям (быт, семья, для себя и т.д.), чтобы понять, какой сфере я уделил больше времени и соблюдается ли баланс.

**US-R4. Мини-статистика за день**  
Я как пользователь хочу видеть в отчёте за день краткую сводку: сколько задач выполнено и по скольким категориям, чтобы одним взглядом оценить продуктивность.

**US-R5. Недельная картина**  
Я как пользователь хочу видеть отчёт «Сделано за неделю», чтобы оценить, сколько дел сделано за 7 дней, и не терять общую картину прогресса.

**US-R6. Недельная группировка по дням**  
Я как пользователь хочу, чтобы в отчёте за неделю задачи были сгруппированы по дням (пн, вт, ср…), чтобы понимать, в какие дни я был активнее.

**US-R7. Недельная статистика по категориям**  
Я как пользователь хочу видеть в отчёте за неделю распределение задач по категориям (сколько в быту, сколько в семье и т.д.), чтобы отслеживать гармонию и баланс между сферами.

**US-R8. Визуальная ясность**  
Я как пользователь хочу, чтобы отчёты были оформлены читаемо: эмодзи категорий, разделители, заголовки — чтобы было приятно смотреть и легко сканировать взглядом.

**US-R9. Пустой отчёт как поддержка**  
Я как пользователь хочу, чтобы при отсутствии выполненных задач бот писал короткое поддерживающее сообщение (например: «Сегодня пока ни одной задачи не отмечено. План на сегодня ещё можно выполнить!»), а не сухо «Нет выполненных задач».

## 2. Категории задач

### 2.1 Пользовательская история (расширенная)

**Исходная:**  
Я как пользователь хочу, чтобы задачи, которые я вношу, присваивались категориям, чтобы их можно было группировать и в дальнейшем отслеживать прогресс и не забывать, всё ли гармонично — уделяю ли достаточно внимания быту, себе, семье и т.д.

**Дополнения (продуктовый менеджер):**

**US-C1. Автоматическое присвоение при добавлении**  
Я как пользователь хочу, чтобы при добавлении задачи бот сам определял категорию по тексту, чтобы мне не приходилось каждый раз выбирать её вручную.

**US-C2. Ручная смена категории**  
Я как пользователь хочу иметь возможность изменить категорию задачи (например, командой «Смени категорию [задача] на быт» или выбором из списка), если бот ошибся или я хочу переложить её в другую сферу.

**US-C3. Группировка в списках и отчётах**  
Я как пользователь хочу видеть задачи сгруппированными по категориям в списке задач, плане на сегодня и в отчётах, чтобы быстро ориентироваться и оценивать баланс.

**US-C4. Фиксированный, но расширяемый набор**  
Я как пользователь хочу, чтобы список категорий был фиксированным (чтобы не раздувался и не запутывал), но при этом в будущем можно было добавлять свои — например, «Работа», «Учёба» — если потребуется.

**US-C5. Прозрачность и контроль**  
Я как пользователь хочу видеть в подтверждении добавления задачи её категорию («📂 Категория: 🏠 Быт») и понимать, что бот её определил сам, чтобы при ошибке сразу скорректировать.

### 2.2 Архитектура: определение категории по задаче (без LLM)

#### 2.2.1 Фиксированный список категорий (расширяемый)

Стартовый набор из 8 категорий (из PRD), хранящийся в коде/конфиге:

| ID | Emoji | Название | Ключевые слова / триггеры |
|----|-------|----------|---------------------------|
| 1  | 🏠 | Быт / дом | уборка, стирка, мыть, прибрать, пылесос, покупки, продукты, магазин, ремонт, квартиру, дом |
| 2  | 👨‍👩‍👧 | Семья | муж, жена, дочка, сын, мама, папа, дети, семья, бабушка, дедушка, врач, кружок, школа |
| 3  | 💇‍♀️ | Уход / внешность | маникюр, салон, стрижка, эпиляция, массаж, врач, косметолог, процедура |
| 4  | 🌿 | Для себя | отдохнуть, прогулка, чтение, книг, медитация, спорт, зал, йога, хобби |
| 5  | 🎫 | Досуг | билеты, кино, театр, ресторан, поездка, отпуск, развлечения |
| 6  | 📦 | Дела / поручения | документ, оплат, звонок, заказ, зарегистрир, записать, оформить |
| 7  | 🧠 | Большие проекты | ремонт, переезд, запуск, проект, планирование |
| 8  | 🔁 | Регулярные | каждый день, еженедел, раз в неделю, регулярно, рутина |
| 0  | 📝 | Другое | fallback, если ничего не подошло |

#### 2.2.2 Алгоритм присвоения категории

**Вход:** нормализованный текст задачи (без даты/времени, lowercase, лишние пробелы убраны).

**Шаги:**

1. **Проверка на регулярность**  
   Если в тексте есть маркеры повторяемости: «каждый», «ежедневно», «еженедельно», «раз в неделю», «каждую неделю», дни недели в контексте повторения — → **🔁 Регулярные**.

2. **Подсчёт совпадений по ключевым словам**  
   Для каждой категории (кроме «Другое») считается число вхождений её ключевых слов в текст задачи.  
   - Можно использовать: точное вхождение подстроки или стемминг/морфологию (опционально, сложнее).  
   - Простой вариант: `keyword in text` для каждого keyword.

3. **Выбор категории**  
   - Если одна категория имеет максимальный счёт и счёт > 0 → она выбирается.  
   - Если несколько категорий с одинаковым максимальным счётом → приоритет по порядку в списке или выбор «Другое».  
   - Если все счёты = 0 → **📝 Другое**.

4. **Опциональные правила (эвристики)**  
   - Слова «мама», «папа», «муж», «жена», «дети», «дочка», «сын» — сильный вес для **Семья**.  
   - «Купить», «заказать» + (продукты, еда) → **Быт**; + (документ, билет) → **Дела** или **Досуг** по контексту.  
   - «Врач», «врачу» — может быть **Семья** (с ребёнком) или **Уход** (для себя); при неоднозначности можно склоняться к **Уход**.

#### 2.2.3 Реализация (псевдокод)

```
CATEGORIES = [
    ("routine", "🔁", "Регулярные", ["каждый", "ежедневно", "еженедельно", ...]),
    ("home", "🏠", "Быт / дом", ["уборка", "стирка", "покупки", ...]),
    ("family", "👨‍👩‍👧", "Семья", ["муж", "дочка", "мама", ...]),
    ...
    ("other", "📝", "Другое", []),  # fallback
]

def assign_category(task_text: str) -> (emoji, name):
    text = normalize(task_text)  # lower, strip, collapse spaces
    for cat_id, emoji, name, keywords in CATEGORIES:
        if cat_id == "other": continue
        if cat_id == "routine" and has_repeat_markers(text):
            return CATEGORIES["routine"][1], CATEGORIES["routine"][2]
    scores = {cat: count_matches(text, cat.keywords) for cat in CATEGORIES}
    best = max(scores, key=scores.get)
    return best.emoji, best.name if scores[best] > 0 else ("📝", "Другое")
```

#### 2.2.4 Расширяемость

- Список `CATEGORIES` вынести в конфиг (JSON/YAML) или таблицу БД.
- Для пользовательских категорий: отдельная таблица `user_categories(user_id, emoji, name, keywords[])`.
- При определении категории: сначала проверять пользовательские, затем системные.
- В настройках: «Добавить категорию» → пользователь вводит название и ключевые слова (или берёт пресет).

### 2.3 Связь с текущим кодом (bot v2)

- В `add_task` уже передаются `category_emoji`, `category_name` (сейчас всегда «Другое»).

- В `db.py`: колонки `category_emoji`, `category_name` уже есть.
- В отчётах (`_format_done_report`) и списках задач (`_format_task_list`) — добавить отображение категории и группировку по ней.
