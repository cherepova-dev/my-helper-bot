# Изменение задачи: пользовательские истории и бэклог

Документ задаёт пользовательские истории и способ реализации функций изменения и переноса задач.

---

## 1. Пользовательские истории

**US-EDIT1. Изменить описание задачи**  
Я как пользователь хочу изменить внесённую задачу, например её описание (текст), чтобы исправить формулировку или опечатку без удаления и повторного добавления задачи.

**US-EDIT2. Перенести задачу по дате или времени**  
Я как пользователь хочу перенести задачу по времени или по дате (на завтра, на пятницу, на 10:00), чтобы скорректировать план без потери самой задачи.

---

## 2. Способ реализации

### 2.1 Распознавание намерения

- **Триггеры изменения текста:** «изменить задачу», «исправить задачу», «переименовать», «задача должна называться», «название задачи», «описание задачи».
- **Триггеры переноса:** «перенести», «перенеси», «переложи», «на завтра», «на пятницу», «на 10:00», «перенеси задачу 3 на завтра».

После распознавания намерения из фразы извлекаются:
- идентификатор задачи (номер из списка или часть названия);
- для изменения: новый текст (всё после маркера или из следующего сообщения);
- для переноса: новая дата и/или время (через существующие `parse_due_date`, `parse_due_time`).

### 2.2 Режим «ожидаю уточнение»

При команде «Изменить задачу» бот может перейти в режим ожидания следующего сообщения:
- «Какую задачу изменить? Напиши номер или часть названия» → следующее сообщение = идентификатор;
- «Напиши новый текст задачи» → следующее сообщение = новый текст.

Либо всё в одной фразе: «Изменить задачу 2 на Купить хлеб».

### 2.3 Вызов слоя данных

- **Изменение текста:** `db.update_task(task_id, user_id, text=new_text)`.
- **Перенос:** `db.update_task(task_id, user_id, due_date=..., due_time=...)`.  
Для рутин при переносе даты можно обновлять только `due_date`/`due_time` если они используются, либо трактовать «перенести рутину» как смену дня недели: `update_task(..., repeat_day=new_day)`.

### 2.4 Ответ пользователю

После успешного изменения или переноса отправить подтверждение:  
«Задача обновлена: «Новый текст» / «Перенесена на завтра в 10:00»».

## 3. Бэклог (функциональные задачи)

| # | Задача | Приоритет |
|---|--------|-----------|
| EDIT-F1 | Распознавание фраз «изменить задачу», «исправить задачу», «переименовать» и извлечение номера/названия и нового текста | Высокий |
| EDIT-F2 | Распознавание «перенеси задачу N на дату/время» и извлечение номера задачи и новой даты/времени | Высокий |
| EDIT-F3 | Режим ожидания: после «Изменить задачу» — следующее сообщение = номер или название, затем (опционально) следующее = новый текст | Средний |
| EDIT-F4 | Вызов `db.update_task` для текста и для даты/времени; ответ пользователю с подтверждением | Высокий |
| EDIT-F5 | Для рутин: «перенести на пятницу» трактовать как смена дня недели (repeat_day) | Средний |

## 4. Связь с кодом

- **db.py:** уже есть `update_task(task_id, user_id, **kwargs)` с полями `text`, `due_date`, `due_time`, `is_routine`, `repeat_day`.
- **bot_v2.py:** добавить обработку текста/голоса с маркерами изменения и переноса; при необходимости — состояния `_awaiting_edit_task`, `_awaiting_edit_new_text`; вызов `update_task` и форматирование ответа.
- **task_parsing.py:** при необходимости — функции `starts_with_edit_marker(text)`, `extract_edit_target(text)` (номер/название + новый текст), `starts_with_reschedule_marker(text)`, `extract_reschedule_target(text)` (номер + дата/время).
