# PRD: AI-ассистент «Личный помощник»

**Продукт:** Telegram-бот — личный AI-ассистент  
**Цель:** Помогать пользователю управлять личными задачами, бытом, семьёй, заботой о себе и планированием — без давления, в комфортном темпе.  
**Платформа:** Telegram (текст + голос)  
**AI-модель:** DeepSeek (бесплатная) или аналог  
**Хостинг:** Render (Python) → позднее VPS + Docker  

---

## 1. Видение продукта

Бот — это **заботливый помощник**, а не менеджер и не начальник. Он:
- принимает задачи в свободной форме (текст или голос);
- сам структурирует, приоритизирует и группирует;
- мягко напоминает, но не давит;
- помогает планировать день так, чтобы пользователь не терял себя в делах.

**Тон общения:** дружелюбный, спокойный, поддерживающий. Без формальностей, без давления, без «ты опять не сделала».

---

## 2. Пользователь

Один пользователь (владелец бота). Бот знает контекст: семья (муж, дочь, мама), привычки, предпочтения. Контекст накапливается со временем.

---

## 3. Категории задач (по умолчанию)

| Иконка | Категория | Примеры |
|--------|-----------|---------|
| 🏠 | Быт / дом / рутины | Уборка, стирка, покупки, ремонт |
| 👨‍👩‍👧 | Семья | Дела с мужем, дочь (кружки, врачи), мама |
| 💇‍♀️ | Уход / внешность | Салон, маникюр, врачи, процедуры |
| 🌿 | Для себя | Отдых, прогулки, чтение, медитация |
| 🎫 | Досуг с мужем | Билеты, рестораны, поездки |
| 📦 | Дела / поручения | Документы, звонки, оплаты |
| 🧠 | Большие проекты | Ремонт, переезд, запуск чего-то |
| 🔁 | Регулярные дела | Еженедельная уборка, полив цветов |

Пользователь может добавлять, переименовывать и удалять категории.

## 4. Онбординг (первый запуск)

При первом `/start` бот приветствует пользователя и коротко объясняет, что умеет. Без «стены текста» — ёмко, с примерами.

### Сценарий первого запуска

```
Привет! Я твой личный помощник. Помогу с задачами, планами и рутиной — без давления и лишнего стресса.

Вот что я умею:

📝 Записывать задачи — просто напиши или надиктуй, я разберусь.
   Например: «Купить продукты завтра» или «Записать дочку к врачу на пятницу»

📋 Показывать список дел — «Что у меня на сегодня?» или «Покажи задачи»

✅ Отмечать сделанное — «Готово: купила продукты»

📊 Приоритизировать — я сам оценю важность и срочность, но ты всегда можешь поправить.

⏰ Напоминать — мягко и вовремя, без давления.

🗂 Категории: 🏠 быт · 👨‍👩‍👧 семья · 💇‍♀️ уход · 🌿 для себя · 🎫 досуг · 📦 дела · 🧠 проекты · 🔁 рутины

Для начала — просто напиши мне свою первую задачу!
```

### Подсказки (Tips)

Бот периодически показывает подсказки в первые дни использования (после каждой 3–5-й задачи), пока пользователь не освоится:

| # | Подсказка |
|---|-----------|
| 1 | 💡 Ты можешь писать задачи голосом — просто отправь голосовое сообщение. |
| 2 | 💡 Напиши «Что на сегодня?» — я покажу план дня с приоритетами. |
| 3 | 💡 Большую задачу можно разбить на шаги. Напиши «Разбей [задачу] на шаги». |
| 4 | 💡 Чтобы отметить дело как сделанное, напиши «Готово: [задача]». |
| 5 | 💡 Я умею работать с датами: «завтра», «в пятницу в 14:00», «через неделю». |
| 6 | 💡 Напиши «Покажи категории» — можно добавить свои или переименовать. |
| 7 | 💡 Если дел накопилось много — попроси: «Выбери 3 самых важных на сегодня». |

Подсказки не повторяются. После показа всех 7 бот больше их не выводит (можно вернуть через `/help`).

## 5. Функции (Features)

### 5.1 Приём задач

**Вход:** текстовое или голосовое сообщение в свободной форме.

**Что делает бот:**
1. Распознаёт задачу (через AI: извлекает суть, дату/время, категорию).
2. Записывает в список задач.
3. Подтверждает: краткое сообщение — что записал, в какую категорию, на когда.
4. Если задача большая (AI оценивает размер ≥ 7) — предлагает разбить на микро-шаги.
5. Если похожая задача уже есть — спрашивает: «У тебя уже есть похожее дело: "…". Объединить или оставить отдельно?»

**Примеры ввода:**
- «Записать дочку к стоматологу на следующей неделе»
- «Купить подарок маме» (без даты → без дедлайна, в общий список)
- «Завтра вечером забрать вещи из химчистки»
- голосовое: «Надо позвонить в управляющую компанию насчёт счётчиков»

### 5.2 Дата и время

1. **Текущее время** — бот знает дату и время по часовому поясу пользователя (задаётся при первом запуске или командой).
2. **Относительное время** — бот понимает:
   - «сегодня», «завтра», «послезавтра»
   - «сегодня вечером», «утром», «днём»
   - «через 2 дня», «через неделю», «в пятницу»
   - «на следующей неделе»
3. **Явная дата/время** — «15 марта в 14:00», «01.04 в 10:00».
4. **Без даты** — задача попадает в общий список (бэклог) без дедлайна.
5. **Отметка «сделано»** — если пользователь пишет «сделала [задачу]» или «готово», бот отмечает задачу как выполненную с текущей датой/временем.

### 5.3 Приоритизация

Бот автоматически оценивает каждую задачу по 4 параметрам (1–10):

| Параметр | Что означает |
|----------|-------------|
| **Ценность** | Насколько важно для жизни / семьи / себя |
| **Срочность** | Есть ли дедлайн, насколько близко |
| **Риски** | Что будет, если не сделать (последствия) |
| **Размер** | Сколько времени/сил займёт (1 = 5 минут, 10 = эпик на недели) |

**Формула приоритета:**  
`Приоритет = (Ценность + Срочность + Риски) / Размер`

Чем выше — тем важнее сделать в первую очередь.

Пользователь может скорректировать баллы: «Подними срочность у задачи X до 9».

### 5.4 Планирование дня

**Утренний план** (по расписанию или по запросу):
- Бот показывает задачи на сегодня, сгруппированные по времени и приоритету.
- Формат:

```
Доброе утро! Вот план на сегодня:

⏰ Утро
  1. 🏠 Записать показания счётчиков (приоритет: 8.0)
  2. 💇‍♀️ Маникюр в 11:00

⏰ День
  3. 👨‍👩‍👧 Забрать дочку из школы в 13:30
  4. 📦 Позвонить в УК

⏰ Вечер
  5. 🌿 Прогулка (для себя)

📋 Без времени (если будет окно):
  6. 🏠 Разобрать шкаф (размер: 6)

Хорошего дня!
```

**По запросу:** «Что у меня на сегодня?», «Покажи план», «Что на завтра?»

### 5.5 Напоминания

- Бот напоминает о задачах с дедлайном: за 1 день и за 1 час (настраиваемо).
- Тон мягкий: «Напоминаю: сегодня в 14:00 — маникюр. Всё ок?»
- Если пользователь не отметил задачу как сделанную — бот **не ругает**, а спокойно переносит: «Не успела — ничего страшного. Перенести на завтра?»
- Регулярные задачи (🔁) напоминаются по расписанию (ежедневно, еженедельно и т.д.).

### 5.6 Управление задачами

**Команды / фразы:**
- «Покажи все задачи» / «Список дел»
- «Покажи задачи по категории 🏠»
- «Что на сегодня?» / «Что на эту неделю?»
- «Удали задачу [название]»
- «Перенеси [задачу] на пятницу»
- «Отметь [задачу] как сделанную» / «Готово: [задача]»
- «Разбей [задачу] на шаги»
- «Измени приоритет у [задачи]»

Бот понимает эти команды в свободной форме (через AI), не требует точного синтаксиса.

### 5.7 Контекст и память

- Бот помнит контекст **внутри сессии** (текущий разговор).
- Бот хранит **профиль пользователя**: имя, часовой пояс, список категорий, предпочтения.
- Бот хранит **историю задач** (выполненные и текущие) — в базе данных (SQLite / файл JSON на первом этапе).
- При обращении к AI передаётся: текущий список задач, профиль, последние сообщения → AI «знает» контекст.

### 5.8 Голосовые сообщения

- Бот принимает голосовые сообщения.
- Распознаёт речь в текст (Google Speech Recognition или Whisper).
- Обрабатывает полученный текст как обычное сообщение (задача, команда, вопрос).

## 7. Тон и UX-принципы

1. **Помощник, не начальник.** Бот предлагает, не приказывает.
2. **Без давления.** Не использует фразы: «Ты опять не сделала», «Срочно!», «Дедлайн горит!»
3. **Поддержка.** Если много невыполненных задач: «У тебя накопилось много дел. Может, выберем 3 самых важных на сегодня? Остальное подождёт.»
4. **Для себя.** Периодически (раз в 2–3 дня) бот мягко напоминает: «Ты давно ничего не планировала для себя. Может, прогулка или книга?»
5. **Лаконичность.** Ответы короткие. Не «простыни текста», а суть + эмодзи.

## 8. Техническая архитектура

### 8.1 Компоненты

```
Telegram ──▶ bot.py (python-telegram-bot)
                │
                ├──▶ AI-модуль (DeepSeek API)
                │      Обработка свободного текста:
                │      - извлечение задачи, даты, категории
                │      - генерация ответа
                │      - приоритизация
                │
                ├──▶ Хранилище (SQLite / JSON)
                │      - задачи (текст, категория, дата, приоритет, статус)
                │      - профиль пользователя
                │      - история
                │
                └──▶ Планировщик (APScheduler или cron)
                       - утренний план
                       - напоминания
```

### 8.2 AI-модель

- **DeepSeek** (бесплатный API) — основной вариант.
- Fallback: другая бесплатная модель (Groq, Together AI и т.п.).
- AI получает:
  - системный промпт (роль бота, тон, правила)
  - текущий список задач (краткий)
  - профиль пользователя
  - последние N сообщений (контекст сессии)
- AI возвращает:
  - структурированный ответ (JSON): задача, категория, дата, приоритет
  - текстовый ответ пользователю

### 8.3 Хранение данных

**MVP (этап 1):** SQLite (файл на диске).  
**Позже:** PostgreSQL (если переедем на VPS или понадобится масштабирование).

**Таблицы:**

```
users:
  id, telegram_id, name, timezone, settings_json, created_at

tasks:
  id, user_id, text, category, status (active/done/cancelled),
  priority_value, priority_urgency, priority_risk, priority_size,
  priority_score, due_date, due_time, repeat_rule,
  parent_task_id (для подзадач), created_at, completed_at

categories:
  id, user_id, name, emoji, sort_order

messages:
  id, user_id, role (user/assistant), text, created_at
```

### 8.4 Зависимости (Python)

```
python-telegram-bot>=21.0    # Telegram Bot API
openai>=1.0                  # DeepSeek API (совместим с OpenAI SDK)
apscheduler>=3.10            # Планировщик (напоминания, утренний план)
python-dateutil>=2.8         # Парсинг дат
```

## 11. Ограничения и риски

| Риск | Митигация |
|------|-----------|
| DeepSeek может быть недоступен | Fallback на другую бесплатную модель |
| AI неправильно извлекает задачу | Бот переспрашивает: «Правильно ли я понял: ...?» |

| Голос не распознаётся | Fallback: «Не удалось распознать. Попробуй текстом.» |
| Бесплатные лимиты API | Мониторить; при необходимости — кэширование ответов AI |

## 12. План технических улучшений (миграция инфраструктуры)

### Этап 1 — Текущий (бесплатный)
- **Хостинг:** Render Free + UptimeRobot (пинг каждые 5 мин)
- **БД:** Supabase PostgreSQL (бесплатный план, 500 МБ)
- **AI:** Groq Free (llama-3.3-70b-versatile + whisper-large-v3-turbo)
- **Ограничения:** сервис может перезапускаться; 750 ч/мес

### Этап 2 — Переезд на Koyeb (бесплатный, без засыпания)
- **Когда:** если Render станет нестабильным или засыпания будут мешать
- **Что даёт:** бесплатный план без засыпания, деплой через GitHub
- **Усилия:** ~1 час (создать аккаунт, подключить репо, задать env vars)
- **БД:** остаётся Supabase

### Этап 3 — VPS (платный, полный контроль)
- **Когда:** готовность платить ~150-300 руб/мес; несколько сервисов
- **Рекомендация:** Timeweb Cloud или Aeza — российские провайдеры, оплата российской картой
- **Что даёт:** Docker, несколько сервисов, своя БД PostgreSQL, полный контроль
- **Усилия:** ~2-3 часа (настройка сервера, Docker, перенос БД)
- **БД:** PostgreSQL на VPS (миграция данных из Supabase)
